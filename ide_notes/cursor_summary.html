📖 Cursor 是什么？

🎯 核心定义

Cursor 是一个基于 VSCode 开发的 AI 辅助编程工具，集成了 GPT-4、Claude 3.5 等先进大语言模型，提供智能代码生成、修改和调试功能。

🛠️ 技术角度

  • 基于 VSCode 架构
  • 集成多种 LLM 模型
  • 本地代码索引技术
  • RAG 检索增强生成

📦 产品角度

  • AI 辅助编程 IDE
  • 多模态交互界面
  • 跨文件代码理解
  • 实时协作编程

💼 商业角度

  • 提升开发效率
  • 降低学习成本
  • 减少代码错误
  • 加速项目交付

⚔️ Cursor vs GitHub Copilot

对比维度 Cursor GitHub Copilot
架构基础 独立 IDE（基于 VSCode） VSCode 插件
上下文理解 全项目代码索引 当前文件 + GitHub API
代码修改能力 跨文件编辑、Agent 模式 主要是代码补全
交互方式 Chat、Composer、Command-K 主要是 Tab 补全
模型选择 多模型支持（GPT-4、Claude等） 主要基于 Codex

⌨️ 核心快捷键

快捷键 功能 使用场景 特点
Tab 智能代码补全 编写代码时 自动预测下一步编辑
Ctrl+K 代码编辑提示栏 局部代码修改 快速生成和修改代码
Ctrl+L AI 聊天功能 代码问答和解释 支持上下文引用
Ctrl+I Composer 模式 跨文件项目编辑 多文件同时修改
Ctrl+Shift+L 添加代码到上下文 选中代码后使用 快速引用代码片段
Ctrl+Enter Codebase 搜索 全项目代码查询 基于 RAG 的智能搜索

🔗 上下文引用系统

📊 引用范围层级

Code < Files ≈ Git < Folders < Codebase

从小到大的上下文范围设计，让开发者根据问题复杂度灵活选择最合适的上下文信息。

🌐 外部信息引用

  • @Docs - 技术文档库

⚙️ Cursor Rules 配置

🌐 全局规则示例

# 在 Cursor 设置 > AI 规则中配置 - 所有新代码使用 TypeScript - 遵循代码整洁之道原则 - 优先使用 async/await 而不是回调 - 编写完善的错误处理机制 - 使用函数式编程模式 - 保持代码简洁和可读性

📁 项目规则示例 (.cursor/rules/react.mdc)

# 文件模式：*.tsx, *.ts ## React 开发规范 - 使用函数组件替代类组件 - 实现完整的 TypeScript 类型定义 - 遵循 React Hooks 最佳实践 - 使用 React.memo 优化性能 ## 样式规范 - 使用 Tailwind CSS 进行样式开发 - 遵循移动优先的响应式设计 - 保持组件样式的一致性 @file ../tsconfig.json @file ../tailwind.config.js

🏗️ 架构与底层原理

🧠 核心技术架构

  • 基础架构：基于 Electron + VSCode 核心
  • AI 集成：多 LLM 模型支持（GPT-4、Claude、o1等）
  • 代码理解：本地向量化索引 + RAG 检索
  • 上下文管理：智能上下文窗口管理
  • 实时协作：云端同步 + 本地缓存

🔄 工作流程

  1. 代码索引和向量化

  3. 上下文检索和组装
  4. LLM 推理和生成
  5. 代码应用和验证

🎉 总结

🌟 Cursor 的核心价值

Cursor 不仅仅是一个代码补全工具，而是一个完整的 AI 辅助编程生态系统。它通过深度理解代码上下文、提供多模态交互方式、支持端到端的开发流程，真正实现了人机协作编程的愿景。
